# 任务方案简报 — 阶段 0.5 输出模板（唯一真源）

> **何时必须先做 0.5、能否豁免、禁止项**以 **`cursor/rules/task-design-brief-gate.mdc`**（`alwaysApply`）为准。  
> **流程顺序**以 `cursor/rules/xiezhi-workflow.mdc` 为准。本文仅提供 Step1–4 的**可复制模板与检查项**。  
> **路径**：本文件位于 `skills` 仓库的 `cursor/references/task-design-brief.md`（与 `cursor/rules` 同级）。

## 核心原则

先定结构再写代码。数据结构是地基，接口契约是边界，耦合分析是风控，Task 拆分是执行计划。

## 触发条件（摘要）

**完整触发与豁免**以 **`cursor/rules/task-design-brief-gate.mdc`** 为准（该文件每次对话自动注入）。典型需做 0.5：新 Phase/Task、新接口、新模块、跨模块、表结构驱动开发。

## 四步设计流程（顺序不可颠倒）

```
Step 1: 数据结构设计 ──► Step 2: 接口契约定义 ──► Step 3: 模块耦合分析 ──► Step 4: Task 拆分
```

每一步输出后**必须等待用户确认**，确认后才可进入下一步。

## 执行规则

1. 四步顺序不可颠倒；每步等待用户确认。
2. 设计文档是契约；偏差须显式提出。
3. 简化：无新接口可跳过 Step2 说明；无跨模块跳过 Step3 说明；单 Task 跳过 Step4。**Step1 永远不可跳过**。
4. 四步合计建议 **60–100 行**以内。

## 与主流程的衔接

任一 Step 未通过时，禁止进入下一 Step 或獬豸阶段 1（编码）。

